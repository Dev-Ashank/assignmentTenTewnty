# Contest Participation System

A production-grade backend API for a contest participation platform built with **NestJS**, **PostgreSQL**, and **TypeORM**. Users can register, participate in contests, answer questions, and compete on leaderboards.

## Features

- **Role-based access control** — Admin, VIP, User, and Guest roles with different permissions
- **JWT authentication** — Secure token-based auth with Passport
- **Contest management** — Create and manage contests with configurable access levels (Normal/VIP)
- **Multiple question types** — Single-select, multi-select, and true/false questions
- **Automatic scoring** — Correct answers earn points, incorrect answers have no penalty
- **Leaderboard** — Ranked by score with standard competition ranking (ties get same rank)
- **Prize awarding** — Admin can award prizes to top scorers
- **User history** — Track past participations, in-progress contests, and prizes won
- **Rate limiting** — Prevents API abuse (configurable per endpoint)
- **Swagger API docs** — Auto-generated interactive documentation
- **Global error handling** — Consistent error responses across all endpoints

## Tech Stack

## Setup & Installation

### 1. Clone the repository

```bash
git clone <repository-url>
cd twentytwentyassignment
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up PostgreSQL

Create the database:

```bash
# Using psql
psql -U postgres -c "CREATE DATABASE contest_db;"
```

Or using pgAdmin / any PostgreSQL GUI client.

### 4. Configure environment

Copy the example `.env` file and adjust values if needed:

```bash
cp .env.example .env
```

Default `.env` values:

```env
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=postgres
DB_NAME=contest_db
JWT_SECRET=your-super-secret-jwt-key-change-in-production
JWT_EXPIRY=24h
PORT=3000
THROTTLE_TTL=60000
THROTTLE_LIMIT=100
```

### 5. Start the server

```bash
# Development mode (with hot reload)
npm run start:dev

# Production mode
npm run build
npm run start:prod
```

The server starts at `http://localhost:3000`.

### 6. Seed sample data (optional)

```bash
npm run seed:db
```

This creates sample users, contests, and questions for testing:
- **admin** / password123 (Admin role)
- **vip_user** / password123 (VIP role)
- **normal_user** / password123 (User role)

## Database Schema

The database auto-syncs from TypeORM entities on startup. Here's the schema:

### Tables

| Table | Description |
|-------|-------------|
| `users` | User accounts with roles (admin, vip, user, guest) |
| `contests` | Contest definitions with access levels and timing |
| `questions` | Questions belonging to contests (JSONB for options/answers) |
| `participations` | User participation records with status and scores |
| `answers` | Individual answers to questions |
| `leaderboard` | Ranked scores per contest |
| `prizes` | Awarded prizes linked to users and contests |

### Entity Relationships

```
Users ──┬──< Participations >──┬── Contests
        │                      │
        ├──< LeaderboardEntries >──┤
        │                          │
        └──< Prizes >──────────────┘
                                    │
                            Questions >──┘
                                    │
        Participations ──< Answers >──┘
```

### SQL Schema (auto-generated by TypeORM)

To generate a SQL dump of the schema:

```bash
pg_dump -U postgres -s contest_db > schema.sql
```

## API Documentation

### Interactive Swagger Docs

After starting the server, visit: **http://localhost:3000/api/docs**

### API Endpoints Summary

#### Authentication
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/auth/register` | Public | Register a new user |
| POST | `/auth/login` | Public | Login and get JWT token |
| GET | `/auth/profile` | JWT | Get current user profile |

#### Users
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| PATCH | `/users/:id/role` | Admin | Update user role |

#### Contests
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/contests` | Admin | Create a contest |
| GET | `/contests` | Public | List contests (filtered by role) |
| GET | `/contests/:id` | Public | Get contest details with questions |
| PUT | `/contests/:id` | Admin | Update a contest |
| DELETE | `/contests/:id` | Admin | Delete a contest |

#### Questions
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/contests/:contestId/questions` | Admin | Add question to contest |
| GET | `/contests/:contestId/questions` | Public | Get questions (answers hidden) |
| PUT | `/contests/:contestId/questions/:id` | Admin | Update question |
| DELETE | `/contests/:contestId/questions/:id` | Admin | Delete question |

#### Participation
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/contests/:contestId/participate` | User/VIP | Join a contest |
| POST | `/contests/:contestId/answers` | User/VIP | Submit an answer |
| POST | `/contests/:contestId/submit` | User/VIP | Finish contest and get score |

#### Leaderboard
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| GET | `/contests/:contestId/leaderboard` | JWT | View contest rankings |

#### Prizes
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/contests/:contestId/prizes/award` | Admin | Award prize to winner |

#### User History
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| GET | `/users/me/history` | JWT | Past participations |
| GET | `/users/me/history/in-progress` | JWT | In-progress contests |
| GET | `/users/me/prizes` | JWT | Prizes won |

## Usage Examples

### 1. Register a new user

```bash
curl -X POST http://localhost:3000/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "john_doe",
    "email": "john@example.com",
    "password": "SecureP@ss123"
  }'
```

### 2. Login

```bash
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "john_doe",
    "password": "SecureP@ss123"
  }'
```

Use the returned `accessToken` in subsequent requests.

### 3. Join a contest

```bash
curl -X POST http://localhost:3000/contests/{contestId}/participate \
  -H "Authorization: Bearer {your_token}"
```

### 4. Submit an answer

```bash
curl -X POST http://localhost:3000/contests/{contestId}/answers \
  -H "Authorization: Bearer {your_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "questionId": "{questionId}",
    "selectedAnswers": ["Paris"]
  }'
```

### 5. Finish the contest

```bash
curl -X POST http://localhost:3000/contests/{contestId}/submit \
  -H "Authorization: Bearer {your_token}"
```

## Postman Collection

Import the included `postman_collection.json` file into Postman for a ready-to-use collection with all endpoints and example requests.

**Setup in Postman:**
1. Import `postman_collection.json`
2. Set the `base_url` variable to `http://localhost:3000`
3. Use the login endpoint first to get a JWT token
4. The token is automatically saved and used for subsequent requests

```json
  "path": "/api/endpoint"
}
```
